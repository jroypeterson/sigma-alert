# sigma-alert — Project Instructions

## Git sync
After making code changes, always commit and push to GitHub (`origin master`) before ending the conversation. Also push after completing a significant change or when it has been a while since the last push — don't let unpushed work accumulate. Use descriptive commit messages.

## "Let's finish" workflow
When the user says "let's finish", "we're done", "wrap up", or anything similar that signals the end of a working session, perform this checklist before responding:

1. **Save memory** — write any new feedback, project, user, or reference memories from the session that aren't already captured. Update or remove stale entries.
2. **Update documentation** — refresh `README.md`, `CLAUDE.md`, and any other relevant doc files for the changes made during the session. Don't let docs drift behind the code.
3. **Run tests** — `python -m pytest tests/ -q` must pass before pushing.
4. **Commit and push** — stage relevant files, write a descriptive commit message, push to `origin master`. Include the sibling `Coverage Manager` repo if its files were touched in this session.
5. **Surface anything skipped** — if there are unrelated uncommitted changes, surface them and ask before touching them. Never silently commit or revert work the user didn't explicitly authorize.

## Project structure
- `scripts/sigma_screener.py` — Main screener: data download, distribution math, alert tier logic, Slack formatting
- `scripts/sync_watchlist.py` — Merges `sources/*.txt` into `watchlist.txt` (does NOT touch `ticker_metadata.json`)
- `sources/` — Per-segment ticker lists (hc_services, medtech, sp500, sector_etfs)
- `watchlist.txt` — Combined deduped ticker list, generated by sync_watchlist.py
- `ticker_metadata.json` — `{TICKER: {name, sector, subsector}}` lookup, **owned by Coverage Manager** (written via its weekly-build sigma-export step). Do NOT regenerate it from CI — the runner has no access to the Coverage Manager CSV
- `cache/distribution_cache.json` — Per-ticker μ/σ from EOD run, used by morning open run to skip full history download
- `cache/missing_metadata.json` — Written by EOD run when watchlist tickers are missing from `ticker_metadata.json` (or have a blank name). Coverage Manager's weekly `sigma_export` reads this from the sibling clone and surfaces the gaps. File is deleted automatically when there are no gaps. Committed by `sigma-close.yml` alongside the distribution cache
- `.github/workflows/` — sigma-open, sigma-midday, sigma-close (cron-fired), sync-watchlist, sigma-watchdog (recovers dropped runs)
- `tests/` — pytest suite

## Sibling projects
- `../Coverage Manager/` — Owns `ticker_metadata.json`. Its weekly-build pipeline writes the file directly into this clone via `reporting/sigma_export.py` and pushes only that single file.

## Key conventions
- All time logic uses `America/New_York` (ET) via `zoneinfo.ZoneInfo`
- yfinance `auto_adjust=True` is used so Close prices reflect splits/dividends
- z-score formula: `z = (today_return - μ) / σ`, where μ/σ come from trailing 252 daily returns
- Alert tiers in Slack output:
  - **2σ+ Moves** — fires on the entire watchlist when `|z| ≥ 2.0`. 3σ+ moves get an inline warning emoji
  - **1σ Moves** — fires only on tickers tagged `Healthcare Services`, `MedTech`, or `PA` in `ticker_metadata.json` when `1.0 ≤ |z| < 2.0`. Sector filter keeps the lower threshold from being noisy
- Within each tier, alerts are grouped into subcategories in fixed order: **Healthcare Services → MedTech → Other/PA → S&P 500** (driven by the `SUBCATEGORIES` list in `sigma_screener.py`). A ticker matching multiple categories is rendered once per category — intentional duplication. 2σ alerts outside all four buckets (e.g. a Biopharma name not in the S&P 500) are **dropped** rather than shown in an "Other" catch-all. S&P 500 membership comes from `sources/sp500.txt` via `load_sp500_set()` — a static list maintained in this repo, not from Coverage Manager, updated manually after quarterly reconstitutions
- Alerts within each tier sorted by signed z-score descending (biggest gainers top, biggest losers bottom)
- Alert lines render as `🟩/🟥 *TICKER* (Short name) | z = +X.XX | +Y.YY% | $PRICE` — sector is implied by the subcategory header, so it's not repeated per line. 52-week hi/lo lines still carry `[Sector]` inline. Short name via `short_company_name()` (which strips both space-separated and glued suffixes like `AptarGroup` → `Aptar`)
- Cron schedules use EST-aligned UTC offsets so they always fire after the relevant market event year-round (1 hour late during EDT, but never early)
- `Sigma Watchdog` workflow runs hourly during weekday market hours, detects dropped scheduled runs, and triggers them via `workflow_dispatch`. Posts a Slack heartbeat only when it actually recovers something
- Slack webhook is stored as `SLACK_WEBHOOK` GitHub Actions secret (note: just `SLACK_WEBHOOK`, not `SLACK_WEBHOOK_URL`)
