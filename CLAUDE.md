# sigma-alert — Project Instructions

## Git sync
After making code changes, always commit and push to GitHub (`origin master`) before ending the conversation. Also push after completing a significant change or when it has been a while since the last push — don't let unpushed work accumulate. Use descriptive commit messages.

## "Let's finish" workflow
When the user says "let's finish", "we're done", "wrap up", or anything similar that signals the end of a working session, perform this checklist before responding:

1. **Save memory** — write any new feedback, project, user, or reference memories from the session that aren't already captured. Update or remove stale entries.
2. **Update documentation** — refresh `README.md`, `CLAUDE.md`, and any other relevant doc files for the changes made during the session. Don't let docs drift behind the code.
3. **Run tests** — `python -m pytest tests/ -q` must pass before pushing.
4. **Commit and push** — stage relevant files, write a descriptive commit message, push to `origin master`. Include the sibling `Coverage Manager` repo if its files were touched in this session.
5. **Surface anything skipped** — if there are unrelated uncommitted changes, surface them and ask before touching them. Never silently commit or revert work the user didn't explicitly authorize.

## Project structure
- `scripts/sigma_screener.py` — Main screener: data download, distribution math, alert tier logic, Slack formatting
- `scripts/sync_watchlist.py` — Merges `sources/*.txt` into `watchlist.txt` (does NOT touch `ticker_metadata.json`)
- `scripts/refresh_sp500.py` — Quarterly S&P 500 reconstitution refresh from Wikipedia. Writes BOTH `sources/sp500.txt` (ticker list) and `sources/sp500_names.json` (ticker→company name map, used by the screener as a name fallback)
- `scripts/weekly_skip_report.py` — Reads `cache/skip_log.json` and posts a trailing-7-day skip digest to the `#status-reports` Slack channel every Friday. Standalone — does not require Coverage Manager
- `sources/` — Per-segment ticker lists (hc_services, medtech, sp500, index_etfs, sector_etfs) plus name-fallback JSONs
- `sources/sp500_names.json` — `{TICKER: "Company Name"}` from Wikipedia, used at startup to fill name gaps for S&P 500 tickers that Coverage Manager doesn't maintain (CM only covers Healthcare/MedTech/PA). Regenerated by `refresh_sp500.py` on every run (even when ticker set is unchanged, to catch renames like Facebook→Meta)
- `sources/index_etfs.txt` — Broad-market index ETFs (SPYM, DIA, QQQ). Render at the top of the **Index & Sector Returns** Slack section
- `sources/sector_etfs.txt` — SPDR Select Sector ETFs (XLE, XLF, XLV, etc.). Render below the indices in the same section
- `sources/etf_names.json` — `{TICKER: friendly name}` for both index and sector ETFs (Coverage Manager doesn't track ETFs, so display names live here). Merged into the metadata copy at startup the same way `sp500_names.json` is
- `watchlist.txt` — Combined deduped ticker list, generated by sync_watchlist.py
- `ticker_metadata.json` — `{TICKER: {name, sector, subsector}}` lookup, **owned by Coverage Manager** (written via its weekly-build sigma-export step). Do NOT regenerate it from CI — the runner has no access to the Coverage Manager CSV. The screener deep-copies this dict at startup and merges `sp500_names.json` into the copy for display purposes; the original is kept intact for `write_missing_metadata_flag` so CM still sees true gaps
- `core_watchlist.json` — `{TICKER: {buy_price, target_price, date_added, notes, name, sector, subsector}}`, **owned by Coverage Manager** (pushed in the same commit as `ticker_metadata.json`). The screener loads this into a ticker set and tags matching alerts with `on_watchlist=True`; those render under a "Core Watchlist" subcategory at the top of each tier. File is optional — missing/empty file just means no watchlist section renders
- `cache/distribution_cache.json` — Per-ticker `{mu, sigma, sample_size, high_52w, low_52w}` from EOD run, used by morning open run to skip full history download. `high_52w`/`low_52w` cached so the cached-open path can render the 52w range; existing entries missing these fields degrade gracefully until the next close run refreshes the cache
- `cache/missing_metadata.json` — Written by EOD run when watchlist tickers are missing from `ticker_metadata.json` (or have a blank name). Coverage Manager's weekly `sigma_export` reads this from the sibling clone and surfaces the gaps. File is deleted automatically when there are no gaps. Committed by `sigma-close.yml` alongside the distribution cache
- `cache/skip_log.json` — Per-run skip events written by EOD close runs, trimmed to a trailing 30-day window. Schema: `{"runs": [{"date", "mode", "skipped": [{"ticker", "reason"}]}]}`. Reasons emitted: `insufficient_history`, `distribution_nan`, `fallback_insufficient`, `fallback_exception`. Consumed by `scripts/weekly_skip_report.py` for the Friday digest. Committed by `sigma-close.yml`
- `.github/workflows/` — sigma-open, sigma-midday, sigma-close (cron-fired), sync-watchlist, sigma-watchdog (recovers dropped runs), sigma-weekly-skip-report (Friday 22:30 UTC, posts skip digest to `#status-reports`)
- `tests/` — pytest suite

## Sibling projects
- `../Coverage Manager/` — Owns `ticker_metadata.json`. Its weekly-build pipeline writes the file directly into this clone via `reporting/sigma_export.py` and pushes only that single file.

## Key conventions
- All time logic uses `America/New_York` (ET) via `zoneinfo.ZoneInfo`
- yfinance `auto_adjust=True` is used so Close prices reflect splits/dividends
- z-score formula: `z = (today_return - μ) / σ`, where μ/σ come from trailing 252 daily returns
- Alert tiers in Slack output:
  - **2σ+ Moves** — fires on the entire watchlist when `|z| ≥ 2.0`. 3σ+ moves get an inline warning emoji
  - **1σ Moves** — fires only on tickers tagged `Healthcare Services`, `MedTech`, or `PA` in `ticker_metadata.json` when `1.0 ≤ |z| < 2.0`. Sector filter keeps the lower threshold from being noisy
- Within each tier, alerts are grouped into subcategories in fixed order: **Healthcare Services → MedTech → Other/PA → S&P 500** (driven by the `SUBCATEGORIES` list in `sigma_screener.py`). A ticker matching multiple categories is rendered once per category — intentional duplication. 2σ alerts outside all four buckets (e.g. a Biopharma name not in the S&P 500) are **dropped** rather than shown in an "Other" catch-all. S&P 500 membership comes from `sources/sp500.txt` via `load_sp500_set()` — a static list maintained in this repo, not from Coverage Manager, updated manually after quarterly reconstitutions
- Alerts within each tier sorted by signed z-score descending (biggest gainers top, biggest losers bottom)
- Alert lines render as `🟩/🟥 *TICKER* (Short name) | +Y.YY% | z = +X.XX | $PRICE | 52w: $LOW - $HIGH` — % change comes before z-score. Sector is implied by the subcategory header, not repeated per line. 52-week hi/lo lines (the new-high/new-low list in the close digest) still carry `[Sector]` inline. Short name via `short_company_name()` (which strips both space-separated and glued suffixes like `AptarGroup` → `Aptar`). Names for S&P 500 tickers not in Coverage Manager's metadata come from `sources/sp500_names.json` (Wikipedia fallback)
- Below the alert tiers, every Slack message renders an **Index & Sector Returns** section. Indices (SPYM/DIA/QQQ from `sources/index_etfs.txt`) appear first under an `_Indices_` sub-header sorted by z-score, then SPDR sector ETFs (`sources/sector_etfs.txt`) under `_Sectors_`, also sorted by z-score. The screener unions both ETF sets into the download list at startup so the section still renders even if a watchlist sync (Coverage Manager's, in particular) drops the ETFs from `watchlist.txt`. Data reuses the same `_process_ticker_full` output that drives alerts — no extra downloads
- Cron schedules use EST-aligned UTC offsets so they always fire after the relevant market event year-round (1 hour late during EDT, but never early)
- `Sigma Watchdog` workflow runs hourly during weekday market hours, detects dropped scheduled runs, and triggers them via `workflow_dispatch`. Posts a Slack heartbeat only when it actually recovers something
- Slack webhooks — stored as GitHub Actions secrets:
  - `SLACK_WEBHOOK` — main sigma-alert channel (`#stock-price-alerts`). Used by the sigma screener for open/midday/close digests
  - `SLACK_STATUS_REPORTS_WEBHOOK` — `#status-reports` channel. Used only by `scripts/weekly_skip_report.py`
